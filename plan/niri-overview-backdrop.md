# Niri Overview Backdrop — Dithered Background

## Goal

When entering niri's overview mode (Super+A), the background between and
behind workspace thumbnails should use the garden palette — ideally with the
signature Bayer dither pattern.

## Problem

Niri composites all layer shell surfaces as part of each workspace's content.
During overview, everything (Background through Overlay layers) gets zoomed
with the workspace thumbnails. There is no layer shell surface that stays
fullscreen behind the overview.

The space between zoomed-out workspace thumbnails is filled by niri's own
`background-color` setting, which is compositor-level rendering outside the
Wayland layer shell pipeline.

### What we tried

- `WlrLayer.Bottom` — invisible; sits behind niri's own background fill.
- `WlrLayer.Overlay` — visible briefly during the zoom transition, then
  zoomed out with the workspace content. Not persistent.

## Options

### Option A: Palette-colored `background-color` (solid, no dither)

Generate a niri KDL include file that sets `background-color` to the
palette's `base-deep`. Niri live-reloads includes on file change, so it
follows palette switches via `garden-themes apply`.

```kdl
// garden-colors.kdl — generated by garden-themes
background-color "#252d3b"
```

This is the simplest approach and can be bundled with the niri generator
for border colors (which is also pending).

### Option B: Always-on dithered wallpaper

Put the `DitherOverlay` on `WlrLayer.Background`, always visible. During
normal tiling use it's hidden behind windows. During overview it shows as
the workspace background *within* each thumbnail (not between them).

### Option C: Both

Solid palette `background-color` between thumbnails + dithered wallpaper
inside them.

## Existing Infrastructure

- `OverviewBackdrop.qml` already exists at `_qml/overlays/OverviewBackdrop.qml`
  and is wired into `shell.qml`. Currently non-functional for the reasons above.
- `NiriAdapter.qml` already handles `OverviewOpenedOrClosed` events and
  exposes an `overviewOpen` property.
- `DitherOverlay.qml` has density presets ("dense", "light", "lock").

## Related: Niri Generator

A niri KDL generator is needed regardless (for border color live switching).
The overview background-color would be part of the same generated file:

```kdl
// garden-colors.kdl — generated by garden-themes
background-color "#252d3b"
layout {
    border {
        active-color "#4a5568"
        inactive-color "#3a4456"
    }
}
window-rule {
    match title="frontier"
    match title="andes"
    match title="summit"
    border { active-color "#c4796b"; }
}
window-rule {
    match title="dgx-"
    border { active-color "#c9b88c"; }
}
window-rule {
    match title="homelab"
    border { active-color "#7c9a7c"; }
}
```

Fern's `niri.nix` would add `include "garden-colors.kdl"` and the apply
command would write this file + trigger `niri msg action do-screen-transition`
for a smooth visual switch.
